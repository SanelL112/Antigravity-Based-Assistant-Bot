# AI Developer Context

**If you are an AI Agent reading this file**, welcome to the Personal Assistant Bot codebase! The user has asked you to help maintain, debug, or extend this system. This file provides the critical architectural context you need to safely edit this project.

## Core Philosophy
1. **Local Only**: This system strictly uses a local Ollama model (`llama3.2`). **DO NOT** import OpenAI, Anthropic, or any other external API packages for text generation. 
2. **Asynchronous Architecture**: The Telegram bot (`main.py`) runs asynchronously. Do not introduce blocking, synchronous logic (like massive `PyPDF2` extractions or synchronous API requests) into the main event loop. Heavy logic belongs in the `scrapers/` background scripts.
3. **No Direct Database**: This system uses raw markdown files (`mega_index.md`, `curated_brain.md`) and JSON files as its database. Do not attempt to install PostgreSQL or SQLite unless specifically requested.

## Codebase Map

### 1. `main.py`
The entry point. It hosts the `python-telegram-bot` instance and sets up the internal task scheduler (`JobQueue`).
- Handles incoming messages, images, and voice notes.
- Uses `ai_processor.py` to route queries to Ollama.
- Contains the `watchdog_check` (runs every 30 mins) and `morning_digest` (runs daily at 7:00 AM).

### 2. `scrapers/` Directory (The Engine)
This is where all data ingestion occurs.
- `google_scraper.py`: Contains API logic to fetch Google Docs, Classroom assignments, and Gmail. Critical logic: `download_drive_file` natively exports Google Docs to PDF so they aren't rejected as "binary".
- `canvas_scraper.py`: Hooks into the Canvas API to pull assignments.
- `historical_export.py`: A script designed to run *once* (or manually) to scrape months of historical data and drop it into `offline_archive/delta_export.txt`. It processes Google Drive items using `corpora="allDrives"` to ensure shared documents are found.
- `nightly_indexer.py`: A heavy background process that reads `delta_export.txt`, feeds it in chunks to Ollama to generate summaries, and appends the results to `mega_index.md`.
- `nightly_processor.py`: Triggers at 2:00 AM to download new PDFs from Classroom and extract their text into `source_cache/pdf_exports.txt`.

### 3. Knowledge Files (The Brain)
- `mega_index.md`: A summarized catalog of the user's data. Written by the offline indexer.
- `curated_brain.md`: An exhaustive, append-only, date-stamped timeline of everything the bot sees or does. It contains NO summaries, just raw events.
- `bot_context.txt`: A tightly compressed, token-optimized text file generated by `compile_context.py` every 30 minutes. This file is physically injected into the `llama3.2` system prompt in `main.py` to give the bot its memory.

### 4. Important Integration Scripts
- `morning_digest.py`: Queries the Google and Canvas APIs to compile a morning briefing, then uses Ollama to format it nicely before sending via Telegram.
- `notion_client.py`: Integrates with the user's Notion database. The watchdog uses this to push new assignments directly to Notion.
- `mega_study_builder.py`: A specialized agentic script that reads `pdf_exports.txt` and `mega_index.md` to generate multi-section study guides.

## Common Pitfalls & Rules for Editing
- **Server Environment**: The user runs this on a Debian server. If you need to restart the bot after changes, use `sudo systemctl restart bot.service`.
- **Google Drive Permissions**: If you query Google Drive, ALWAYS use `supportsAllDrives=True`, `includeItemsFromAllDrives=True`, and `corpora="allDrives"` to prevent missing shared docs.
- **Ollama Hanging**: If Ollama hangs at 100% CPU due to a massive prompt chunk, restarting the `ollama` systemd service clears the hung process.
- **Context Window**: Be highly mindful of the 8K context limit on `llama3.2`. Always use `compile_context.py` to trim memory files before injecting them into the prompt.
